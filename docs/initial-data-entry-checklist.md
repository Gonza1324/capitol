# Capitol Hub - Checklist de carga inicial

Usar esta checklist para cargar los primeros datos reales de Capitol Hub de forma ordenada.

## Antes de empezar

- [ ] Confirmar que el cliente debe cargarse en Capitol Hub.
- [ ] Definir quien sera responsable interno de la carga.
- [ ] Revisar si el cliente ya existe para evitar duplicados.
- [ ] Confirmar que la informacion a cargar esta validada.
- [ ] Acordar criterio de nombres para clientes, contactos, stakeholders y documentos.

## Por cada cliente

### Datos generales

- [ ] Nombre comercial o nombre de referencia.
- [ ] Razon social, si corresponde.
- [ ] CUIT o identificacion fiscal, si corresponde.
- [ ] Estado: active, prospect, paused, former, potential o archived.
- [ ] Tipo de cliente.
- [ ] Descripcion breve.
- [ ] Perfil estrategico.
- [ ] Fecha de inicio.
- [ ] Fecha de finalizacion, si aplica.
- [ ] Nivel de confidencialidad.
- [ ] Website.
- [ ] Link a carpeta externa, si existe.
- [ ] Notas generales relevantes.

### Rubros e issues

- [ ] Rubros asociados.
- [ ] Issues asociados.
- [ ] Prioridad de cada issue: high, medium o low.
- [ ] Fecha de inicio del issue, si aplica.
- [ ] Fecha de fin del issue, si aplica.

### Responsables internos

- [ ] Usuario responsable principal.
- [ ] Otros usuarios asignados.
- [ ] Descripcion simple de responsabilidad por usuario.

### Contactos

- [ ] Contacto principal.
- [ ] Contactos secundarios relevantes.
- [ ] Cargo y organizacion/area.
- [ ] Email.
- [ ] WhatsApp o telefono.
- [ ] LinkedIn, si aplica.
- [ ] Rol en la relacion.
- [ ] Observaciones relevantes.
- [ ] Marcar inactivos los contactos que no correspondan.

### Tareas activas

- [ ] Tareas abiertas actuales.
- [ ] Responsable/s.
- [ ] Prioridad.
- [ ] Fecha limite.
- [ ] Cliente asociado.
- [ ] Comentarios iniciales si ayudan a entender contexto.

### Interacciones recientes

- [ ] Ultimas calls o reuniones relevantes.
- [ ] Emails importantes.
- [ ] Presentaciones o almuerzos relevantes.
- [ ] Participantes internos.
- [ ] Participantes externos.
- [ ] Resumen.
- [ ] Decisiones.
- [ ] Riesgos.
- [ ] Proximos pasos.
- [ ] Tareas derivadas.

### Reportes recientes

- [ ] Reportes enviados recientemente.
- [ ] Tipo de reporte.
- [ ] Estado.
- [ ] Tema.
- [ ] Responsable.
- [ ] Fecha de envio.
- [ ] Destinatarios.
- [ ] Link externo o documento asociado.
- [ ] Notas.

### Alertas recientes

- [ ] Alertas enviadas recientemente.
- [ ] Categoria.
- [ ] Urgencia.
- [ ] Canal usado.
- [ ] Responsable.
- [ ] Clientes asociados.
- [ ] Rubros e issues asociados.
- [ ] Destinatarios.
- [ ] Link o archivo asociado.
- [ ] Notas.

### Stakeholders relacionados

- [ ] Stakeholders clave vinculados al cliente.
- [ ] Tipo.
- [ ] Organizacion.
- [ ] Cargo.
- [ ] Jurisdiccion.
- [ ] Area de influencia.
- [ ] Nivel de influencia.
- [ ] Postura.
- [ ] Nivel de sensibilidad.
- [ ] Observaciones.

### Documentos y links

- [ ] Contratos o documentos base, si corresponde.
- [ ] Reportes o presentaciones relevantes.
- [ ] Links externos a Drive, Docs, PDFs u otros recursos.
- [ ] Entidad asociada correcta.
- [ ] Tipo de documento.
- [ ] Descripcion clara.

## Recomendaciones de carga

- Evitar duplicados antes de crear clientes, contactos o stakeholders.
- Usar nombres consistentes: no alternar abreviaciones sin criterio.
- Cargar solo informacion validada.
- No subir documentos altamente sensibles hasta confirmar permisos y criterio interno.
- Usar links externos cuando el documento ya esta ordenado en otro repositorio.
- Priorizar primero clientes activos y tareas abiertas.
- Registrar interacciones recientes que expliquen decisiones y proximos pasos.
- Usar el historial del cliente para revisar si la carga quedo comprensible.

## Control final por cliente

- [ ] La ficha del cliente tiene datos generales completos.
- [ ] Rubros e issues estan asociados.
- [ ] Responsables internos estan asignados.
- [ ] Contacto principal cargado.
- [ ] Tareas abiertas cargadas.
- [ ] Interacciones recientes cargadas.
- [ ] Reportes y alertas recientes cargados.
- [ ] Stakeholders relevantes cargados.
- [ ] Documentos o links importantes cargados.
- [ ] El historial del cliente muestra una historia coherente de trabajo.
